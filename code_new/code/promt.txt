1. NONDOMINATED SORTING (Table 2 approach):
   
   def dominates(sol_a, sol_b, objectives_a, objectives_b) -> bool:
       """
       sol_a dominates sol_b nếu:
       - ∀i: fi(a) ≤ fi(b)  
       - ∃j: fj(a) < fj(b)
       """
   
   def nondominated_sort(population, objectives) -> List[List[int]]:
       """
       Return: fronts = [front1_indices, front2_indices, ...]
       front1 = Pareto front (best)
       Dùng fast nondominated sort O(MN²) hoặc
       divide-and-conquer O(NlogM-1 N) nếu M<=3
       """

2. CROWDING DISTANCE (Table 2):
   
   def crowding_distance(front_indices, objectives) -> np.ndarray:
       """
       Với mỗi objective fj:
       - Sort solutions trong front theo fj
       - Boundary solutions: cd = 1 (capped từ infinity)
       - Các solution còn lại:
         cdj(Xl) = (fj(Xl+1) - fj(Xl-1)) / (fmax_j - fmin_j)
       Total: cd(Xl) = sum(cdj(Xl) for j in objectives)
       """

3. SELECTION based on rank + crowding:
   
   def select_best(population, objectives, n_select) -> List[int]:
       """
       Ưu tiên: rank thấp hơn → crowding distance cao hơn
       Giữ lại nSol solutions tốt nhất
       """

4. GBEST SELECTION:
   
   def select_gbest(pareto_front, objectives, crowding_distances) -> int:
       """
       Chọn gbest từ ψ1 (Pareto front)
       Dựa trên crowding distance để đảm bảo diversity
       Dùng roulette wheel selection với probability ∝ crowding distance
       """

## 📦 PROMPT 5 — A*-Based Search (ABS)

```
Implement ABS local search mechanism theo Section 4.5 và Tables 3-5:

CLASS ABSearch:
    
    def __init__(self, instance, nABS=0.2):
        self.instance = instance
        self.nABS = nABS  # probability trigger ABS
    
    def should_apply(self) -> bool:
        """Random trigger: apply if random() < nABS"""
    
    def select_routes_to_remove(self, solution) -> List[int]:
        """
        Eq(18): Selection probability cho mỗi route k
        pk = exp(-10 * nCusk / max(nCusk))
        Pk = pk / sum(pk)
        
        Chọn max(1, round(|τ|/3)) routes
        Ưu tiên routes có ÍT customers (dễ consolidate)
        Dùng roulette wheel selection
        """
    
    def cal_vcost(self, customer, current_route, current_load) -> float:
        """
        Table 4: Feasibility check
        v1: capacity check → inf nếu exceed
        v2: time window check cho customer → inf nếu violate tL
        v3: time window check cho depot → inf nếu không về kịp
        Return: inf nếu bất kỳ vi = inf, else 0
        """
    
    def cal_hcost(self, customer, open_customers, current_route) -> int:
        """
        Table 5: Heuristic cost - ước tính số customers có thể thêm
        Duyệt qua remaining customers theo hfcj = dcj + tcj
        dcj = distance to next customer
        tcj = tL_cj / max(tL) (normalized deadline)
        Count customers có thể serve hợp lệ
        Return: h (count)
        """
    
    def build_route(self, unassigned, current_solution) -> List[int]:
        """
        Table 3: Main ABS procedure
        
        STOPPING CRITERIA (3 conditions):
        1. open set rỗng
        2. validcus rỗng (không có customer feasible)
        3. makespan >= avgmsp (cân bằng workload)
        
        SCORING composite criterion f:
        gci = gcurrent + distance(current, ci)
        gsc_i = gci / max(gci)  [normalized g-cost]
        hsc_i = (max(hci) - hci) / max(hci)  [inverted h-cost]
        fci = gsci + hsci
        
        Dùng ROULETTE WHEEL selection trên f
        """
    
    def apply(self, solution) -> Solution:
        """
        Main ABS application:
        1. Select routes to remove
        2. Extract customers → unasscus
        3. Rebuild routes với build_route()
        4. Repeat until unasscus empty or max vehicles reached
        Return: new solution
        """
```

```

CLASS iNSSSO:
    
    PARAMETERS (từ ANOVA optimal - Table 6):
    - nSol = 100          # population size
    - Cw = 0.99           # probability current solution
    - Cg = 0.95           # probability gbest
    - nABS = 0.2          # ABS trigger probability
    - tRun = 60           # time limit (seconds)
    
    def initialize_population(self) -> List[Solution]:
        """
        Section 4.2: Initialization
        X1: LKH-derived solution (route set → integer → real-number)
        X2...XnSol: random real vectors in [0,1]
        
        LKH conversion process:
        Step 1: Define separator set s = {|C|+1, ..., nVar}
        Step 2: Convert route set → integer code Z
        Step 3: Convert Z → real-number X via random ρ mapping
        """
    
    def update_solution(self, Xi, G, t) -> Solution:
        """
        Eq(2): iNSSSO update rule (pBest REMOVED so với SSO gốc)
        
        for each variable j:
            ρ ~ Uniform(0,1)
            if ρ ∈ (0, Cg]:     xij = gj      (gbest)
            elif ρ ∈ (Cg, Cw]:  xij = xi_prev  (current)
            else:                xij = random   (exploration)
        """
    
    def run(self, instance) -> Tuple[List[Solution], dict]:
        """
        Main loop theo Fig. 1:
        
        INITIALIZATION:
        1. Initialize X0 với LKH + random
        2. Evaluate fitness (f1, f2, f3)
        3. Nondominated sort → fronts ψ1...ψk
        4. Crowding distance calculation
        
        EVOLUTION LOOP (until tCur > tRun):
        for i in range(nSol):
            if random() < nABS:
                Yi = ABS(Xi)           # local search branch
            else:
                gbest = select_gbest(ψ1)
                Yi = update_solution(Xi, gbest)  # SSO branch
            
            Evaluate Yi
        
        POPULATION UPDATE:
        1. Combine X + Y → merged pool (2*nSol solutions)
        2. Nondominated sort merged pool
        3. Crowding distance calculation
        4. Select nSol best solutions (rank + crowding priority)
        5. t = t + 1
        
        RETURN: Pareto front ψ1, convergence history
        """
    
    def get_pareto_front(self) -> List[Solution]:
        """Return current ψ1 (nondominated solutions)"""

## 📦 PROMPT 7 — Performance Metrics

```
Implement 4 metrics đánh giá theo Section 5.1:

CLASS PerformanceMetrics:

    def coverage(self, P_approx, P_true) -> float:
        """
        Eq(19): Cov(P_approx, P_true)
        = |{v ∈ P_true : ∃v' ∈ P_approx, v' dominates or equals v}|
          / |P_true|
        Range: [0, 1], higher = better
        """
    
    def inverted_generational_distance(self, P_approx, P_true) -> float:
        """
        Eq(20): IGD = sum(d(v, P_approx) for v in P_true) / |P_true|
        d(v, P_approx) = min Euclidean distance từ v đến P_approx
        
        NORMALIZE objectives trước khi tính:
        fi_norm = fi / max(fi in P_true)
        Range: [0, ∞), lower = better
        """
    
    def hypervolume(self, P_approx, reference_point=None) -> float:
        """
        Eq(21): HV = volume of union of hypercubes
        
        Normalize theo nadir point:
        fi_norm = fi / max(fi in P_approx)  [scaled to [0,1]]
        
        Reference point: (1.1, 1.1, 1.1) sau normalization
        Dùng WFG algorithm cho 3 objectives (O(N log N))
        """
    
    def n_nondominated_solutions(self, P_approx) -> int:
        """Nnds = |P_approx| - đơn giản là count"""
    
    def estimate_true_pareto(self, all_runs_solutions) -> List:
        """
        Aggregate tất cả solutions từ tất cả algorithms, tất cả runs
        Apply nondominated sort → lấy front 1
        Đây là P_true ước lượng cho bài toán
        """
    
    def compute_all(self, P_approx, P_true) -> dict:
        """
        Return dict với mean và std của 4 metrics
        {
            'Cov': (mean, std),
            'IGD': (mean, std), 
            'HV':  (mean, std),
            'Nnds': (mean, std)
        }
